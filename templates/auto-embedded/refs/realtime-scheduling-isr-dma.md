# 实时控制调度与 ISR/DMA 架构（控制环频率 · 定时器-ADC-DMA 同步 · 裸机 vs RTOS）

> 本文件在以下情形按需加载：
> - 要规划 MCU 上多个控制环/任务的频率、优先级、时序，避免抖动/丢拍/卡死
> - 配定时器触发 ADC + DMA 同步采样、把控制环放对地方
> - 纠结裸机时间片 vs RTOS
>
> 这是所有控制/估计篇的**执行底座**（电流环/姿态环/采样的"跑在哪、多快、谁优先"）。

---

## 铁律

**控制环的周期必须严格、抖动必须小：用硬件定时器/DMA 把"采样—计算—输出"钉在确定时刻，而不是靠主循环"尽量快"。`dt` 来自定时器配置、不硬编码；ISR 只做必须实时的最小工作，重活下沉。**

- 分频分层：高频环(电流/采样)进硬件触发的 ISR；低频环(速度/位置/规划/通信)进主循环或低优先任务（呼应各控制篇的"频率分工"）。
- ISR 铁律：**短、不阻塞、不做重计算/矩阵求逆/浮点海量**；不在 ISR 里 `printf`/忙等/喂看门狗。

---

## 控制环频率与优先级

| 环/任务 | 典型频率 | 跑在哪 | 优先级 |
|---|---|---|---|
| FOC 电流环 / 高频采样 | 10–40 kHz（=PWM） | PWM 触发的 ADC-ISR | 最高 |
| 姿态/平衡/云台内环 | 0.5–5 kHz | 定时器 ISR | 高 |
| 速度/位置/外环 | 100–1000 Hz | 定时器 ISR 或主循环分频 | 中 |
| 规划/通信/状态机/日志 | 10–100 Hz | 主循环 / 低优先任务 | 低 |

- **内环必须比外环快**（带宽分离，见 `pid-control-reference.md` 串级）；中断优先级也要内高外低，防外环阻塞内环。
- 采样率 ≥ 控制带宽的 4–10×（见 PID 篇离散化）。

---

## 定时器 → ADC → DMA 同步（核心范式）

```
定时器(PWM/TRGO) ──硬件触发──► ADC 在确定时刻采样(对齐 PWM 中心) ──► DMA 自动搬到内存 ──► 转换完成中断
                                                                                         │
                                                       在该 ISR 里跑控制环(用刚搬好的数据)─┘  → 输出 PWM
```

- **定时器触发 ADC**：采样时刻精确、与 PWM 对齐（FOC 采样窗口、避开开关噪声，见 `foc-current-loop.md`），不靠软件轮询。
- **DMA 搬运**：CPU 不参与搬数，多通道一次搬完；用"转换完成/半完成"中断驱动控制。
- **串口**用 **DMA + 空闲中断 + 环形缓冲**收（避免逐字节中断打断控制）；**CAN** 按整帧进硬件 FIFO/mailbox、每帧一次中断（必要时配 DMA，无"空闲中断"概念）。二者都"用中断/缓冲驱动、不阻塞收发"（见 `comms-protocol-bus-reliability.md`）。

## 抖动 / 时序卫生

- **抖动(jitter)** 毁控制：用硬件定时器节拍而非 `HAL_Delay`/主循环计数；高要求场景运行期监测两次 ISR 实际间隔。
- **执行时间预算**：测 ISR 耗时（GPIO 翻转 + 示波器），控制在周期的 30–50% 以内（留余量给抖动/其他中断）。
- **数据一致性**：ISR 与主循环共享数据用 `volatile` + 临界区/双缓冲（别读到半更新的状态）；多字节量原子读或关中断短临界区。
- **计算延迟**：采样→输出有一拍延迟，高带宽环要建模/补偿（见 PID 篇）。

## 裸机时间片 vs RTOS

| | 裸机（超级循环 + 定时器中断） | RTOS（FreeRTOS 等） |
|---|---|---|
| 适用 | 任务少、时序简单、电赛主力 | 多任务、复杂并发、通信栈多 |
| 实时控制环 | 放硬件 ISR（不受调度影响） | 仍建议放 ISR / 最高优先任务 |
| 优点 | 确定、轻、无调度开销 | 解耦、优先级、阻塞式写法 |
| 风险 | 大工程主循环膨胀难维护 | 优先级反转、栈溢出、ISR 里调阻塞 API |

- **即使上 RTOS，硬实时控制环仍放 ISR**，RTOS 任务跑非硬实时的逻辑/通信/规划。
- RTOS 陷阱：任务栈给够（防溢出）、ISR 里只用 `FromISR` API、当心优先级反转（用互斥量优先级继承）。

## 常见陷阱

| 陷阱 | 说明 |
|---|---|
| 控制环放主循环"尽量快" | 周期不定、抖动大；放硬件定时器 ISR |
| `dt` 硬编码 | 调度/分频一改就系统性偏；用定时器周期 |
| ISR 里做重计算/矩阵求逆/printf | 拖爆中断、丢拍；重活下沉主循环 |
| 软件轮询 ADC | 时刻不准、占 CPU；定时器触发 + DMA |
| 逐字节串口中断 | 高速时中断风暴打断控制；DMA+空闲中断 |
| 共享数据无保护 | 读到半更新值；volatile+临界区/双缓冲 |
| 中断优先级内外倒置 | 外环阻塞内环；内高外低 |
| RTOS 栈不足/ISR 调阻塞 API | 栈溢出/崩溃；给够栈、用 FromISR |
