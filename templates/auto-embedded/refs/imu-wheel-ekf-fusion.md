# IMU + 轮速里程计的卡尔曼融合（ESKF，MCU 落地）

> 本文件在以下情形按需加载：
> - `omni-wheel-odometry.md` 的纯航位推算不够用了：打滑严重 / 长程累积 / 需要位置不确定度 / 多源异步
> - 要在 Cortex-M（FPU、无 malloc）上把 IMU 和编码器融合成一个状态估计器
>
> 前置：先读 `.auto-embedded/refs/imu-fusion-filter-selection.md` 确认你确实需要升到这一档；
> 运动学反解见 `.auto-embedded/refs/omni-wheel-odometry.md`；姿态来源见 `.auto-embedded/refs/mahony-ahrs-reference.md`。

---

## 铁律

**融合不创造可观测性，只更好地组合各源、给打滑/噪声加权。**

- 无绝对参考的 yaw，融合后**仍然漂**——ESKF 只是让漂得更慢、并给出协方差，不是免标定免漂的银弹。
- 打滑时编码器是**错误观测**：融合器若无条件信任它，照样被带偏。融合的价值恰在于**能给可疑观测降权/门控**。
- 升到 ESKF 前，IMU 必须已完成轴映射 + 量程 + 零偏标定（`imu-gyroscope-checklist.md`），编码器必须已完成 `COUNTS_PER_MM` 标定与正反解互测（`omni-wheel-odometry.md`）。

> **作用域边界**：本方案**不做纯惯性(SINS)位置双积分**。消费级 MEMS（MPU6050/ICM20602）加计零偏使位置按 `O(t²)` 发散——例：`0.001g ≈ 0.0098 m/s²`，积 1 s 约 0.01 m/s 速度误差、约 0.5 cm 位置误差；约 **5 s 到分米级、10 s 到半米级**，几十秒即不可用。故位置/速度的长期可观靠**轮速 + ZUPT/NHC + 外部观测**，IMU 只做高频姿态 + 短时预测。圆锥/划桨误差补偿（多子样等效旋转矢量）是高动态高精 SINS 的姿态/速度更新才需要，电赛低动态 MEMS 属 overkill、单子样积分足够。

## 为什么用 ESKF（误差状态卡尔曼），不用直接 EKF

| 维度 | 直接 EKF（名义状态做滤波） | ESKF（误差状态做滤波） |
|---|---|---|
| 滤波量 | 大信号（位置/速度/四元数） | **小量误差**（δ），线性化误差小 |
| 姿态表示 | 四元数进协方差，过参数化、易奇异 | 误差用 **3 维 δθ**（流形切空间），协方差最小、无奇异 |
| 归一化 | 每步强行归一化四元数，破坏协方差一致性 | 名义四元数单独积分，误差注入后重置，干净 |
| MCU 友好度 | 一般 | **更好**（误差状态维度小、矩阵小） |

**结论**：IMU 融合场景 ESKF 是事实标准，MCU 上也更省。

## 状态设计（competition 高度，按需裁剪）

**名义状态**（高频 IMU 积分，不进协方差）：

```
p  位置(世界系)     v  速度(世界系)     q  姿态四元数
bg 陀螺零偏         (ba 加计零偏，可选；只有姿态+轮速融合时常可省)
```

**误差状态**（进协方差，维度小是关键）：

```
δx = [ δp(3), δv(3), δθ(3), δbg(3) ]   → 12 维（含 δba 则 15 维）
```

> 轮式车若只融合"姿态 + 轮速 + yaw"，可进一步降维（如只估 yaw 误差 + 平面位置 + 陀螺 z 偏），把状态压到个位数，MCU 上更轻。**先按需裁剪，别一上来就 15 维全量。**

> **初始化**：名义 `q` 别用 `(1,0,0,0)` 起步——上电静止时用单帧定姿（TRIAD/QUEST）解出 `q₀` 并据其设初始协方差 `P₀`，显著缩短收敛、避免开机发散。见 `.auto-embedded/refs/attitude-init-single-frame.md`。

> **安装角（IMU↔车体）**：IMU 几乎不可能完美对正底盘轴，存在固定安装偏航角 `ψ_mount` 时，IMU 传播的速度方向与轮速实测的车体速度方向分属两个车体系，其差是**常值系统偏差、会被 ESKF 当真值吸收**（NHC 尤其依赖它）。两条路径：①低成本小车**离线标定** `ψ_mount` 写常量（把 IMU 测量绕 z 旋 `ψ_mount` 后再与轮速比对）；②要长期鲁棒则把 `ψ_mount` 增广为缓变状态在线估计（NHC/直线行驶工况可激励其可观测性）。轮速刻度系数 `COUNTS_PER_MM` 的漂移见 `omni-wheel-odometry.md`（误差源3 + 直线测距标定），短赛程离线标定写死即可。

## 预测 / 更新 / 注入三步

### 1) 预测（IMU 高频，如 200–1000 Hz）

- 名义状态：用陀螺/加计积分推进 `q, v, p`（陀螺减 `bg`）。
- 误差协方差：`P = F·P·Fᵀ + Q`。`F` 是误差状态的离散转移（来自 IMU 运动学线性化），`Q` 来自 IMU 噪声 + 零偏随机游走。

### 2) 更新（编码器低频，如 50–100 Hz）

- **观测 = 车体速度**：用四轮反解（`omni-wheel-odometry.md`）得到 `(vx, vy)`（必要时 `ωz`），作为对名义速度（旋到车体系）的观测。
- 标准卡尔曼更新：`K = P·Hᵀ(H·P·Hᵀ+R)⁻¹`，算 `δx`，更新 `P`。
- **打滑/野值门控（卡方 NIS，统一各观测口径）**：别只比"新息幅度"——它不按协方差归一，`P` 大（开机/机动）时正常新息被误剔、`P` 小时野值漏网。用归一化新息平方 `NIS = innovᵀ·(H·P·Hᵀ+R)⁻¹·innov`（标量观测即 `innov²/(HPHᵀ+R)`；`S=HPHᵀ+R` 在上一行卡尔曼增益里已算出，几乎零增量）。按卡方阈值**三级处置**：低于下阈→正常更新；下阈~上阈→按 NIS 比值放大 `R` 降权；超上阈→丢弃本次观测。χ² 阈值查表（自由度=观测维：1 维 95%≈3.84、99%≈6.63）。这套门控**统一管 ZUPT/NHC/轮速三类观测**，比各自拍幅度阈值规范。保留"加计-轮速一致性"作为打滑的物理旁证。阈值仍按 `coding-standards.md` §4.1 标定溯源。
- **ZUPT（零速更新）**：检测到整车静止（陀螺方差小 + 加计≈1g）时，注入"速度=0"强观测，顺带收敛 `v` 和 `bg`——MCU 上极廉价、收益大，强烈建议加。
- **NHC 非完整性约束（运动中持续生效的"免费"虚拟观测）**：把"车体系侧向速度 ≈ 0"当虚拟量测（`H` 把名义速度旋到车体系取侧向分量 `v_body,y`，地面车再加垂向 `v_body,z`），观测值=0，`R` 小但非零（容忍轻微侧滑/悬挂）。与 ZUPT 互补：ZUPT 管整车**静止**时全速度=0，NHC 管**正常行驶**时持续压横向/垂向漂。
    - ⚠ **必须按底盘类型门控**：对**差速/阿克曼成立**；对**全向/麦轮不成立**——它们能侧移，`v_body,y` 是真实自由度且已被四轮反解直接测量，强加 `v_y=0` 反而注入错误约束（全向底盘至多约束垂向 `v_z≈0`，意义有限）。
    - 前置：先标定 IMU↔车体安装偏航角（见上方"状态设计·安装角"）；打滑/急转侧滑大时与上方卡方门控联动（放大 `R` 或暂关）。
- **ZIHR（零积分航向速率，ZUPT 的航向版）**：静止判据成立时，除注入 `v=0`，再给"航向变化率=0"（或 `Δyaw=0` 锁存观测），钉住静止期 yaw、强化 `bg_z` 收敛。与 `imu-gyroscope-checklist.md` 的 ZARU 同源（位姿域 `Δyaw=0` 是细微新增）。注意：无磁时它只延缓静止期 yaw 漂、不创造绝对航向（行进中仍漂，回铁律）。
- **静止/平稳检测器（ZUPT/NHC/ZIHR 共用）**：判据 = 滑动窗口内 `‖a‖∈[1g−ε, 1g+ε]` 且 `‖ω‖` 与其方差均低于阈值，连续 `N` 个周期（带滞回防抖）。`ε/N/阈值` 是标定项（`coding-standards.md` §4.1，测量法=静止采样取 3σ）。
    - ⚠ **误触发代价**：低速匀速 ≠ 静止——若单看"速度小"就触发 ZUPT，会把真实速度硬拉到 0，制造系统性**里程短缺**；故静止判据必须用"加计≈1g + 陀螺低"而非速度小。NHC 误用在打滑/腾空时同样有害，须与打滑门控联动关断。

### 3) 注入 + 重置

- 把 `δx` 注入名义状态（`p+=δp`，`v+=δv`，`q ⊗= δq(δθ)`，`bg+=δbg`）。
- **重置误差状态为 0**，并按需做协方差重置（姿态误差注入的雅可比）。

## 数值稳健三件套（最易写错、最易发散，给出可抄公式）

| 项 | 公式 / 做法 | 为什么 |
|---|---|---|
| 协方差对称化 | 每步 `for i,j: P[i][j]=P[j][i]=0.5f*(P[i][j]+P[j][i])` | 浮点误差累积破坏对称性 → 失正定 |
| Joseph form 更新 | `P = (I−KH)·P·(I−KH)ᵀ + K·R·Kᵀ` | 比简化式 `(I−KH)P` 数值更稳、对 K 误差不敏感，强烈优先 |
| 注入后协方差重置 | 姿态块用雅可比 `G_θ = I − 0.5·[δθ]×`，整体 `P = G·P·Gᵀ`（其余块 G=I） | 误差注入名义 q、误差态归零后，协方差须按该非线性重置，否则一致性被破坏 |

> `[v]×` 为反对称（叉乘）矩阵。这三处正是下文两次告警"协方差失正定 → 数值发散"的落点；CMSIS-DSP `arm_mat_*` 可直接搭这些小矩阵运算。

## MCU 落地要点

| 项 | 做法 |
|---|---|
| 内存 | 状态维度小 + **定长矩阵**（`float P[N][N]` 静态），**无 malloc** |
| 矩阵运算 | CMSIS-DSP `arm_mat_*`（F4/F7 FPU），或手写小矩阵（N≤15 时可接受） |
| 频率分工 | IMU 高频只跑预测；编码器/ZUPT 低频跑更新（与 `loop` 分频/定时器对齐） |
| 浮点 | F4F/F7F 用 `float`；**禁止在 ISR 内做矩阵求逆**，更新下沉到主循环任务 |
| 协方差正定 | 每步对称化 `P=(P+Pᵀ)/2`；更新用 **Joseph form** 抗数值发散 |
| 时间戳 | IMU 与编码器**同步对齐**；观测有时延要补偿，否则系统性偏 |

## Q / R 怎么定（不要拍脑袋，也不要照抄）

- **Q**（过程噪声）来自 IMU datasheet：角随机游走（ARW）、速度随机游走（VRW）、零偏不稳定度 → 折算到离散过程噪声。
- **R**（观测噪声）来自编码器量化误差 + 打滑统计 + 轮径标定误差。
- 调参方向：估计太信编码器（打滑被吸收成真值）→ 增大 R / 收紧打滑门控；估计太飘 → 减小 Q 或检查 bias 建模。

> ⚠ Q/R/门控阈值**都是标定参数**，必须带单位+来源+测量方法+适用硬件，禁止跨工程照抄，见 `coding-standards.md` §4.1。

## 端到端定位精度怎么验证

闭合/往返自检（见 `omni-wheel-odometry.md`）能抓符号/标定错，但闭合误差会**掩盖"沿途漂出去又漂回来"的中途偏差**，也不量化漂移率。补两项可量化指标：

- **绝对定位误差**：走已知折线/方阵、经过若干已知锚点，逐点比对估计 `(x,y)` 与卷尺/激光实测的偏差（不是只看终点闭合）。
- **漂移率**：固定速度直线/方阵跑 `T` 秒或 `D` 米，记 `位置误差/路程 [%]` 与 `yaw 漂移率 [°/min]`，作为"是否需升级/加绝对观测"的判据。融合（加 ZUPT/NHC）后这两项应较纯航位推算明显改善。

## 常见陷阱

- **忽略零偏状态**：不估 `bg`，yaw 漂会被 ESKF "学"进位置里，越融越歪。
- **静态 Q/R 一把梭**：不同地面/速度打滑特性不同，至少给打滑门控。
- **协方差失正定**：不对称化 / 不用 Joseph form → 数值发散、估计跳变。
- **坐标系/时间戳错配**：观测在车体系、状态在世界系，旋转矩阵或时延搞错 → 系统性偏。验证"观测→状态"符号一致性见 `control-loop-sign-debug.md` 的"观测-状态符号自检（融合专用）"。
- **指望它修 yaw 漂**：无磁/无视觉时 yaw 仍漂，ESKF 救不了（回到铁律）。

## 何时这还不够（升级出口）

- 需要**长期绝对位姿 / 全局一致** → 加绝对观测：磁力计（yaw）、视觉/VIO、UWB、LiDAR。
- 打滑/接触复杂（足式） → InEKF（见 `imu-fusion-filter-selection.md`）。

> **加绝对观测的融合层次（对接业界术语）**：**松组合** = 把外部模块**已解算的位姿/速度**（UWB 定位结果、VIO 位姿）当观测，简单、模块解耦，多数电赛升级首选（本篇 ESKF 用轮速反解 `(vx,vy)` 做观测本质即松组合）；**紧组合** = 把**原始量测**（UWB 各基站测距、视觉特征、GNSS 伪距）直接进滤波 `H`，少观测/部分遮挡也能用，代价是 `H` 复杂；**深组合** = 与信号跟踪环耦合，嵌入式罕见，了解即可。升级优先沿用松组合。

融合到这一步，轮速里程计退化为"高频预测源"，由绝对观测做校正——这才是鲁棒定位的终态。
