# 单帧确定性定姿（Wahba：TRIAD / QUEST / Davenport-q / SVD）— 滤波器初始对准

> 本文件在以下情形按需加载：
> - 递推滤波器（Mahony / EKF / ESKF / MEKF / VQF）**初值差、开机前几秒发散**，想"开机即对准"
> - 静止/准静止时要从**一帧**加计(+磁)**直接闭式**解出姿态，而不是迭代收敛
> - 给卡尔曼类滤波器提供初始四元数 `q₀` 与初始协方差 `P₀`
>
> 这是与递推滤波器**正交**的一族：它做的是"**定姿（determination）**"——用观测矢量代数求姿态；不是"**滤波（filtering）**"——随时间递推。两者配合：单帧定姿给初值，滤波器做时序估计。
> 选型脊柱见 `.auto-embedded/refs/imu-fusion-filter-selection.md`；实现/约定见 `.auto-embedded/refs/mahony-ahrs-reference.md`。

---

## 铁律

**单帧定姿不替代滤波器，它给滤波器一个好的起点。**

- 它解决的是"**初始对准**"：把 `q=(1,0,0,0)` + 上电拉高 Kp 硬等收敛，换成"开机用一帧观测直接算出正确姿态"。
- 它**不抑制噪声、不估零偏、不随时间积分**——这些仍是后续递推滤波器的活。
- 可观测性照旧由**矢量**决定：只有重力一个矢量 → 只能定 roll/pitch（倾角），**yaw 不可观测**；要定 yaw 必须再来一个不平行矢量（磁场）。和滤波篇同一条铁律：缺参考的量谁都定不出来。

---

## 适用 vs 不适用

| 场景 | 用单帧定姿？ |
|---|---|
| 上电静止、要给 Mahony/EKF/ESKF/MEKF/VQF 求初值 q₀ | ✅ 首选 |
| 准静止、纯几何定姿（无需时序平滑） | ✅ |
| 运动中、强动态加速度（加计不再≈纯重力） | ❌ 退回递推滤波器（VQF/Mahony） |
| 要在线零偏、要不确定度传播、要抗磁扰 | ❌ 那是滤波器的活 |

> ⚠ 单帧定姿假设"测到的加计矢量≈重力方向"。**只在静止/准静止时可信**；运动时先判静止（加计模长≈1g 且陀螺方差小）再触发，否则用错矢量会给出错的初值。

## 第 0 步：钉死参考矢量与坐标系（与解算篇同一纪律）

定姿是"机体系观测矢量"与"世界系参考矢量"对齐求旋转，动手前先钉死：

| 项 | 必须明确 | 示例 |
|---|---|---|
| 世界系 | NED 还是 ENU | ENU：z 轴朝上 |
| 重力参考 `a_r` | 世界系里重力（或其反向）方向 | ENU 静止时加计测得"反重力"≈ `[0,0,1]` |
| 磁参考 `m_r` | 世界系里地磁方向（含倾角） | 用当地磁倾角，或只取水平分量定北 |
| 四元数约定 | w-first/Hamilton、机体↔世界方向 | 与 `mahony-ahrs-reference.md` 一致 |

> 机体系观测：`a_b` = 归一化加计读数，`m_b` = 归一化磁力计读数（**先做椭球标定**，见 `imu-gyroscope-checklist.md` 检查项 8）。

---

## TRIAD（两矢量，最简，MCU 默认）

最轻量的解：用两个不平行矢量构造两组正交三元组，旋转矩阵一步求出。**让更可信的矢量（重力）当锚**，磁场只用来定与重力垂直的方向（消除磁倾角/水平度量误差对倾角的污染）。

```c
// 前提：a_b,m_b（机体系）、a_r,m_r（世界系）均已归一化。
// 输出 R：机体→世界 旋转矩阵；重力 a 为主锚，磁 m 仅用于其正交分量。
static void triad(const float a_b[3], const float m_b[3],
                  const float a_r[3], const float m_r[3], float R[3][3]) {
    float t1b[3] = { a_b[0], a_b[1], a_b[2] };          // 锚 = 重力方向
    float t2b[3]; cross(a_b, m_b, t2b); normalize(t2b); // 垂直于重力-磁平面
    float t3b[3]; cross(t1b, t2b, t3b);                 // 补成右手正交基
    float t1r[3] = { a_r[0], a_r[1], a_r[2] };
    float t2r[3]; cross(a_r, m_r, t2r); normalize(t2r);
    float t3r[3]; cross(t1r, t2r, t3r);
    // R = [t1r t2r t3r] · [t1b t2b t3b]^T
    for (int i = 0; i < 3; i++)
        for (int j = 0; j < 3; j++)
            R[i][j] = t1r[i]*t1b[j] + t2r[i]*t2b[j] + t3r[i]*t3b[j];
}
// 然后 R → 四元数（标准 DCM→quat），作为滤波器初值 q₀。
```

- **优点**：O(1)、仅几个叉乘 + 归一化，M0+/M4F 都跑得动；无迭代、无矩阵分解。
- **局限**：只用两个矢量、对噪声不做加权最优；噪声大或有多矢量时用下面的 QUEST/SVD。

---

## QUEST / Davenport-q / SVD（多矢量加权最优，Wahba 问题）

要"带权重的最优解"（多个矢量、各自噪声不同）时，解 **Wahba 问题**：

```
min_R  Σ wᵢ · ‖ b̂ᵢ − R·r̂ᵢ ‖²     // b̂ᵢ 机体系观测、r̂ᵢ 世界系参考、wᵢ 权重
```

- **Davenport-q**：构造姿态廓矩阵 `B = Σ wᵢ b̂ᵢ r̂ᵢᵀ` 与 4×4 对称矩阵 `K`，**最优四元数 = K 最大特征值对应的特征向量**。数学基准解。
- **QUEST**：不显式求特征向量，用牛顿迭代解特征值（初值 `λ≈Σwᵢ`）再回代，比通用特征分解快——卫星定姿经典。
- **SVD（Markley）**：`B = U·S·Vᵀ`，`R = U·diag(1,1,det(U)det(V))·Vᵀ`。**数值最稳**（处理近共线矢量不退化），代价是要 3×3 SVD。

**权重常识**：加计通常比磁力计可信 → `w_acc > w_mag`；磁扰大时进一步压低 `w_mag`。

> MCU 取舍：**初始对准用 TRIAD 基本够**。仅当多矢量、强噪声、或要数值最稳时上 SVD（可离线/低频跑，不必每周期）。QUEST 是"想要最优又嫌特征分解重"的折中。

## 6 轴（只有加计，无磁力计）

只有重力一个矢量 → **只能定 roll/pitch**，yaw 不可观测（这是物理限制，不是算法问题）：

```
roll  = atan2( a_y, a_z )
pitch = atan2(-a_x, sqrt(a_y² + a_z²) )   // 单矢量倾角解，非 TRIAD
yaw   = 0（任意；待磁力计/编码器/视觉给绝对参考）
```

由 (roll, pitch, yaw=0) 转四元数即得初值。yaw 的绝对参考缺失问题同 `mahony-ahrs-reference.md`。

> **器件等级：MEMS 为何不能"自对准"求北**。高精度陀螺（零偏 ≪ 地球自转率 15°/h）能在静基座感知地球自转分量、解析求出真北（罗经法 gyrocompassing），无需磁/外部参考即可定 yaw。但消费级 MEMS（MPU6050/ICM20602 零偏数十~数千 °/h）远远淹没 15°/h，**物理上做不到**——这正是本栈"无磁 yaw 必漂、yaw 绝对初值只能靠磁力计/双天线/已知朝向"的根因。可与 `imu-gyroscope-checklist.md` 的 Allan 方差（零偏不稳定度）交叉印证。

## 接入：替代"q=(1,0,0,0) + 高 Kp 硬收敛"

```
上电 → 等待静止（|a|≈1g 且陀螺方差小，1–2s）
     → 采一帧 a_b（6 轴）或 a_b+m_b（9 轴）
     → TRIAD/单矢量倾角 解出 q₀
     → 用 q₀ 初始化滤波器名义状态；据观测噪声设初始协方差 P₀
     → 进入正常递推（Mahony 用常规 Kp，不必再拉高硬收敛）
```

对 Mahony：直接把 `q0..q3` 设为 `q₀`，省去"上电 Kp=5 加速收敛"那段。对 EKF/ESKF/MEKF：`q₀` 作名义姿态、`P₀` 作初始协方差（见 `imu-wheel-ekf-fusion.md` 初始化）。

## 陷阱

| 陷阱 | 说明 |
|---|---|
| 运动时定姿 | 加计含运动加速度 ≠ 纯重力 → 初值错。必须先判静止再触发 |
| 磁力计没标定 | 硬/软铁未校 → 椭球畸变 → 航向初值偏。先做检查项 8 |
| 两矢量近平行 | 重力与磁场几乎同向（高磁倾角区）→ 叉乘退化。TRIAD 病态，改 SVD |
| 参考矢量约定不一致 | `a_r/m_r` 的 NED/ENU、四元数 w-first/方向与解算篇不一致 → 初值整体错 |
| 把它当滤波器 | 单帧定姿不抑噪、不估偏、不积分；它只负责"起点"，时序估计仍要滤波器 |
