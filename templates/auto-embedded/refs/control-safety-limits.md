# 控制安全 / 限幅 / 故障处理（限流限矩 · 堵转 · 软限位 · 急停 · 看门狗 · 无扰切换）

> 本文件在以下情形按需加载：
> - 给电机/执行器闭环加**安全红线**：限流限矩、堵转、过流过温、软限位、急停、故障态
> - 模式/控制器切换抖一下（无扰切换）、上电冲击、看门狗复位后失控
>
> 控制安全是**红线不是锦上添花**：失效要进**安全态**而非失控。现场完整流程见 `.auto-embedded/modes/competition.md`（看门狗/故障恢复段）；本篇是控制域的方法参考。增益/阈值溯源见 `coding-standards.md` §4.1。

---

## 铁律

**任何执行器闭环都要能"安全地失败"：检测到异常→进受控的安全态（输出归零/coast/刹车 + 告警），而不是放任失控。所有保护阈值都是标定项（带单位+来源+测量方法+适用硬件），留余量、加滞回、防误触发。**

---

## 检查项 1：限流 / 限矩（最常用）

- **限矩 = 限 `Iq*`**：转矩 ∝ Iq，给定限幅 = 转矩上限（保护机械/电机/电源）。串级里这天然就是速度环输出的限幅点。
- **限流 = 限输出电压/占空 + 监测相电流**：电流环 R 限幅 + 母线电流上限。
- **热折返(foldback)**：温度升高时按比例压低电流上限（如只随温度调，保持响应；见 FOC 温补思路），冷热机性能一致、防烧。
- 取值经验：连续限在额定内，峰值限在 ~70–80% 堵转电流附近留余量（具体按电机/驱动手册标定）。

---

## 检查项 2：堵转检测（FOC 下尤其要单独做）

**判据**：转速≈0（陀螺/编码器无变化）**且** 电流/Iq 持续高（> 1.2–2× 空载）连续 N 个周期 → 进故障。

- ⚠ **FOC 下堵转未必触发硬件 OCP**：堵转时电流环会把 Iq 顶到限幅但**不一定超过 OCP 阈值**，电机却在持续发热——所以**必须有"高电流 + 近零速 + 持续时长"的软逻辑**，不能只靠 OCP。
- **忽略上电/启动冲击**：起步浪涌电流大，加 50–100ms 延迟或启动豁免，防误判。
- 编码器版：有速度指令但编码器连续 N 周期读数不变 → 堵转/断线（呼应 competition.md）。
- 命中后：截流/进 ERROR + 告警，定时重试或人工复位。

---

## 检查项 3：过流 / 过压 / 欠压 / 过温保护

- **双层**：硬件（电流采样 + 比较器 / 栅驱 DESAT，**快、ns~µs 级**直接封管）+ 软件（ADC 阈值判断，慢但灵活、可分级）。
- 阈值经验（标定）：相/母线电流 > ~1.2× 额定；母线电压 > ~1.1× 或 < 欠压；温度 > ~80–85℃。
- 过压要配**制动斩波/泄放电阻**（能量回馈母线泵升，急停/反接时尤甚）。
- 太紧会误触发、太松起不到保护——留余量 + 滞回。

---

## 检查项 4：软限位 + 硬限位

- **软限位**：位置接近上下限时**先减速、到限即停**（位置环目标钳到 [min,max] + 接近段降速），比硬撞机械限位优。带滞回防抖。
- **硬限位**：机械挡块 / 限位开关 / 光电，作软限位失效的兜底。
- 上电先**找零/回零**（绝对编码器免找零），否则软限位坐标无基准。

---

## 检查项 5：急停 / 刹车

- **急停优先级最高**：一键/信号 → 立即关 PWM。两种停法：**coast（高阻滑行）** 省事但停得慢；**主动刹车（三相短接/反向）** 停得快但有电流冲击与回馈，按场景选。
- 急停要**绕过正常控制逻辑**（独立硬件/最高优先中断），不能被卡死的主循环挡住。

---

## 检查项 6：看门狗联动

- **IWDG 独立看门狗**必启；**喂狗只在主循环**，禁止在 ISR 里喂（会掩盖主循环卡死）。窗口看门狗可同时抓"跑太快"。
- **复位后进安全态**：电机 PWM 归零、输出关闭、告警，再按状态恢复（用 `.noinit` 区记复位标志，见 `coding-standards.md`）。
- 控制环**心跳**：上位/通信丢失（如遥控信号超时）→ 失控保护（停车/悬停）。

---

## 检查项 7：无扰切换（bumpless）+ 软启动

- **无扰切换**：手动↔自动、控制器/模式切换时，若新控制器积分项与当前输出不匹配会**输出突跳**。做法：切换瞬间**把积分项预载**为"使输出等于当前实际输出"的值（`I ← u_current − (P 项 + D 项)`），输出连续。
- **软启动**：上电/使能时把电压/电流/目标在 100–500ms 内**斜坡**上来，避免冲击。
- 目标突变也用运动规划平滑（见 `motion-profiling-trajectory.md`），别喂阶跃。

---

## 检查项 8：故障状态机

```
NORMAL ──(轻度:接近限位/温升)──► WARN（降增益/限幅/降速，仍运行）
   ▲                                  │
   │(恢复条件满足)                     ▼(严重:堵转/过流/看门狗)
   └──────────── SAFE/ERROR ◄─────────┘   (输出归零/coast/刹车 + 告警 + 记录)
```

- 分级降级优于"非黑即白"：能 WARN 续跑就别直接停。
- 恢复要有**明确条件**（故障消失 + 延时 + 可选人工确认），防抖动反复进出。

## 常见陷阱

| 陷阱 | 说明 |
|---|---|
| 只靠硬件 OCP 防堵转 | FOC 堵转 Iq 饱和未必超 OCP，要软逻辑（高电流+近零速+时长）|
| 保护阈值无余量/无滞回 | 误触发或抖动反复 |
| ISR 里喂狗 | 掩盖主循环卡死 |
| 切换不预载积分 | 输出突跳（bumpless 缺失）|
| 上电不软启动 | 冲击电流/机械冲击 |
| 急停走正常控制路径 | 主循环卡死时急停失效；要独立最高优先 |
| 过压无泄放 | 急停/回馈时母线泵升炸管 |
