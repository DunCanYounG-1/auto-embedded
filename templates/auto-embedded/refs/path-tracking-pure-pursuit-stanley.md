# 路径跟踪：Pure Pursuit / Stanley（横向控制）

> 本文件在以下情形按需加载：
> - 已有位姿估计（惯导/里程计），要让车**跟踪一条路径或航点序列**
> - 在选 Pure Pursuit / Stanley / LQR-MPC 做横向（转向）控制
> - 出现"跟踪抄近道 / 低速画龙 / 高速发散 / 终点附近乱转"等症状
>
> 前置：位姿来源见 `.auto-embedded/refs/omni-wheel-odometry.md` 与 `.auto-embedded/refs/imu-fusion-filter-selection.md`；
> **符号方向陷阱**（本主题高发）见 `.auto-embedded/refs/control-loop-sign-debug.md`；
> 模型法（LQR/MPC）见 `.auto-embedded/refs/lqr-example-bicycle-cornell.md` / `lqr-example-segway.md`。

---

## 铁律：路径跟踪是控制层，不是惯导

```
惯导/状态估计 (IMU+编码器+融合) ──位姿(x,y,yaw)──▶ 路径跟踪 (PP/Stanley/LQR) ──转向/速度──▶ 底盘
```

- 路径跟踪**消费**惯导的位姿，**不修正**估计误差：**位姿不准，跟踪再好也跟不准**。先保证惯导可靠，再调跟踪。
- 本主题**符号陷阱密集**（航向误差极性、横向误差正负、转向极性、坐标系），调参前先按 `control-loop-sign-debug.md` 验证方向——那篇的 Pure Pursuit `κ=2sin(α)/Ld` 符号 bug 案例正是本主题的反面教材。

---

## Pure Pursuit（几何法，前视点追踪）

**核心**：在路径上取车前方距离 `Ld`（前视距离）的一个点，让车沿圆弧驶向它。

```
α  = 车头朝向 与 (车→前视点) 连线 的夹角
κ  = 2·sin(α) / Ld                    // 期望圆弧曲率
δ  = atan( 2·L·sin(α) / Ld )          // 阿克曼转向角，L=轴距（后轴为参考点）
Ld = k_v · v + Ld_min                 // 前视距离随车速自适应
```

| | |
|---|---|
| 优点 | 简单、鲁棒、输出平滑；容忍位姿噪声；参数少 |
| 缺点 | **抄近道**（corner cutting，急弯切内侧）；`Ld` 调参两难——太小→画龙/失稳，太大→切弯且迟钝；恒曲率有稳态误差 |
| 适用 | 中低速、路径平滑、要鲁棒优先于精度 |

## Stanley（前轮反馈，横向误差 + 航向误差）

**核心**（斯坦福 DARPA 用法，参考点在**前轴**）：

```
δ = θ_e + atan( k · e_fa / (v + k_s) )
  θ_e  = 路径切线方向 − 车头朝向        // 航向误差
  e_fa = 前轴到路径的横向误差(cross-track)
  k    = 横向误差增益
  k_s  = softening 常数，防 v→0 时除零/过激
```

| | |
|---|---|
| 优点 | 收敛快、跟踪精度高；横向误差被显式拉回；无前视距离调参 |
| 缺点 | 高速 / 大曲率敏感、易过激；低速需 `k_s` 兜底；对位姿噪声比 PP 敏感；需前轴参考与可靠航向 |
| 适用 | 中低速、要高跟踪精度、路径曲率不极端 |

## 选型表

| 方案 | 机理 | 参数 | 精度 | 鲁棒/容噪 | 何时选 |
|---|---|---|---|---|---|
| Pure Pursuit | 几何前视圆弧 | `Ld`(及速度自适应) | 中（抄近道） | 高 | 默认起步、低速、路径平滑 |
| Stanley | 前轮横向+航向反馈 | `k` / `k_s` | 高 | 中 | 要精度、中低速 |
| LQR / MPC | 模型最优/前瞻约束 | 模型 + 权重 | 高 | 中 | 有模型/算力、要前瞻或约束（见 lqr 示例） |

> 没有特别理由就 **Pure Pursuit 起步**；要精度且位姿够稳再上 Stanley；要最优/前瞻/带约束才上 LQR/MPC。

## 底盘适配（转向角 δ 怎么落到不同底盘）

| 底盘 | 落地 |
|---|---|
| 阿克曼（车型） | PP/Stanley 原生输出转向角 `δ`，直接给舵机 |
| 差速 | 把曲率转角速度 `ωz = v·κ`，再分解为左右轮速差 |
| **全向/麦轮** | 可不走几何跟踪：位置用 pursuit 给平移方向，**航向独立控制**（见 `omni-wheel-odometry.md` 的去参数化锁头状态机）——全向底盘平移与转向解耦，常比硬套 PP 更直接 |

## 共同陷阱

| 陷阱 | 说明 / 对策 |
|---|---|
| **符号** | 航向误差/横向误差/转向极性任一翻转就发散；按 `control-loop-sign-debug.md` 对照实验验证 |
| 参考点 | PP 用后轴、Stanley 用前轴；而上游 `omni-wheel-odometry.md` 的 `(x,y)` 在**车体中心**积分——三者参考点不同，消费前需按轴距/几何做**偏移换算**，否则跟踪系统性偏 |
| 坐标系 | 横向误差、切线方向的世界系/车体系要一致 |
| 航点推进 | 前视点查找 / 航点切换逻辑要稳，避免回跳或跳点 |
| 速度自适应 | `Ld` 不随速度变 → 高速失稳或低速迟钝 |
| 终点处理 | 接近终点 pursuit 前视点失效，需切换到定点（位置闭环）收尾 |
| 参数溯源 | `Ld / k_v / Ld_min / k / k_s` 都要带单位+来源+测量方法+适用硬件，禁裸魔法数字，见 `coding-standards.md` §4.1 |
