# IMU / 陀螺仪模块开发检查清单

> 本文件在 RESEARCH 或 EXECUTE 阶段涉及 MPU6050、ICM20602、BMI088、LSM6DS3 等 IMU 传感器的姿态解算时**必须读取并逐项核对**。

---

## 检查项 1：轴匹配（最高优先级）

互补滤波 / 卡尔曼滤波中，加速度计和陀螺仪的轴必须物理一致。

**判断规则**：`atan2` 用了哪两个加速度计轴，**没被用到的那个轴就是旋转轴**，陀螺仪必须用**同名轴**。

| 加速度计公式 | 倾斜平面 | 旋转轴 | 陀螺仪轴 |
|-------------|---------|--------|---------|
| `atan2(accel_x, accel_z)` | XZ | **Y** | **gyro_y** |
| `atan2(accel_y, accel_z)` | YZ | **X** | **gyro_x** |
| `atan2(accel_x, accel_y)` | XY | **Z** | **gyro_z** |

**常见错误**：变量命名为 `GY` 但实际读取了 `gyro_x` 寄存器，导致轴不匹配。必须追溯到寄存器地址（0x43=gyro_x, 0x45=gyro_y, 0x47=gyro_z）确认。

**验证方法**：手持 PCB 绕预期旋转轴缓慢转动，串口打印陀螺仪原始值，确认有明显变化且符号正确。

## 检查项 2：量程选择

量程决定了传感器能测量的最大范围。**宁大勿小，饱和比精度低危害大得多。**

### 陀螺仪量程

| 应用场景 | 推荐量程 | 灵敏度 |
|---------|---------|--------|
| 平衡车 / 无人机 / 机器人 | **±2000°/s** | 16.4 LSB/°/s |
| 云台 / 手持稳定 | ±500°/s 或 ±1000°/s | 65.5 / 32.8 |
| 缓慢运动追踪 | ±250°/s | 131 |

**平衡车为什么必须用 ±2000°/s**：快速倾倒纠偏时角速度可达 500~1000°/s，低量程直接饱和截断，互补滤波积分跟不上真实角度，一旦快速倾斜就永远算不回来。

### 加速度计量程

| 应用场景 | 推荐量程 | 灵敏度 |
|---------|---------|--------|
| 有电机振动的场合（平衡车、无人机） | **±16g** | 2048 LSB/g |
| 轻微运动（手环、计步） | ±2g 或 ±4g | 16384 / 8192 |

### 量程-灵敏度绑定关系

**改量程必改灵敏度，全局搜索所有引用点。** 常见需要联动修改的位置：
- 互补滤波中的除数（如 `/ 16384.0f` → `/ 2048.0f`）
- 校准函数中的 1g 基准值（Z 轴减去重力）
- 单位转换函数中的 scale 常量
- 头文件中的默认配置结构体

## 检查项 3：DLPF（数字低通滤波器）

DLPF 在芯片内部对原始数据做硬件平滑。

**核心原则**：滤波器引入的延迟不应超过控制周期的 1/5。

| DLPF 值 | 陀螺仪带宽 | 信号延迟 | 适用场景 |
|---------|-----------|---------|---------|
| 0 | 256Hz | **0.98ms** | 平衡车、无人机（延迟最小，可忽略） |
| 1 | 188Hz | 1.9ms | 快速响应 + 轻度滤波 |
| 2 | 98Hz | 2.8ms | 中等响应 |
| 3 | 42Hz | **4.8ms** | 缓慢运动追踪 |
| 6 | 5Hz | **18.6ms** | 极慢运动 |

**平衡车为什么必须用 DLPF=0**：控制周期 5~10ms，DLPF=3 的 4.8ms 延迟意味着 PID 拿到的数据已过时将近一个控制周期。互补滤波本身已能滤除高频噪声（99% 陀螺仪权重），不需要硬件再滤一遍。

**注意**：DLPF 设置还影响采样率分频基准：
- DLPF=0 时，陀螺仪输出率 = 8kHz，`SampleRate = 8kHz / (1 + SMPLRT_DIV)`
- DLPF≠0 时，陀螺仪输出率 = 1kHz，`SampleRate = 1kHz / (1 + SMPLRT_DIV)`

## 检查项 4：互补滤波系数

```c
Angle = Alpha * AngleAcc + (1 - Alpha) * AngleGyro;
```

Alpha 越小 → 越信任陀螺仪（短期精确，长期漂移）
Alpha 越大 → 越信任加速度计（长期稳定，短期噪声大）

**时间常数**：`τ ≈ dt / Alpha`

| 控制周期 dt | Alpha | 时间常数 τ | 效果 |
|------------|-------|-----------|------|
| 10ms | 0.02 | 500ms | 平衡车典型值 |
| 5ms | 0.01 | 500ms | 等效（周期减半，系数也减半） |

**验证方法**：静止时角度值应稳定在 ±0.5° 以内；快速倾斜 90° 后角度应在 0.5 秒内跟上。

## 检查项 5：I2C 通信方式

| 方式 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| 软件 I2C（GPIO 位操作） | 时序可控，无总线锁死 | 占用 CPU | **平衡车首选** |
| 硬件 I2C 外设 | CPU 占用低 | CH32V/STM32 硬件 I2C 有已知锁死问题 | 需加超时和总线恢复 |

**中断中做 I2C 读取时**：确保 I2C 操作不会被更高优先级中断打断导致总线状态错乱。

## 检查项 6：陀螺零偏标定与角速度换算

### 角速度换算：用公式，不要写派生魔法常数

角速度换算系数**从量程灵敏度推导**，不要把算好的小数（如 `0.0010653`）当常数硬写——那样换量程就错且无人看懂。

```
ω[°/s]   = raw_lsb / 灵敏度           // ICM20602 ±2000dps：灵敏度 = 16.4 LSB/°/s
ω[rad/s] = raw_lsb / 灵敏度 × π/180   // 例：raw / 16.4 × 0.017453
```

**反例**：直接写 `gz_rad = raw * 0.0010653f;`（= 1/16.4 × π/180）。系数对，但脱离量程语义；改量程 → 灵敏度变 → 这个常数静默失效。正确：把 `16.4f` 命名为 `ICM20602_GYRO_SENS_LSB_PER_DPS` 并注释量程来源。

### 陀螺零偏：标定，不要用死区"装看不见"

静止时陀螺输出非零（零偏 bias），积分会让 yaw 缓慢漂。两种处理，**优先标定**：

| 做法 | 评价 |
|---|---|
| **上电零偏标定**（推荐起步） | 静止检测（加速度接近 1g 且陀螺方差小）→ 采 N 个样本求均值作 `bias` → 运行期 `ω -= bias`。可周期重标。 |
| **在线 ZARU（零角速度更新）** | 运行中持续检测静止 → 静止时把当前陀螺读数低增益更新进 `bias`（滑动平均），**补偿温漂**，比一次性上电标定更稳。是 ESKF 里 ZUPT 的轻量版，MCU 上极廉价。 |
| 角速度死区（`|ω| < 阈值` 不积分） | **只能作标定后的兜底噪声门限**。单独用它抑漂会**吞掉真实低速旋转**，制造系统性航向误差。用了必须注释标注此副作用。 |

> yaw 绝对参考缺失时（IMU 版无磁力计），零偏标定只能延缓漂移、不能消除——长期绝对 yaw 仍需 9 轴磁力计 / 视觉 / 编码器融合。把死区当"零漂解决方案"是反例，见 `.auto-embedded/refs/case-dcar-control-defects.md`。

## 检查项 7：加速度计零偏 / 尺度标定（六面法）

陀螺要标零偏，加计同样要标**零偏 + 尺度**，否则倾角基准就偏。误差模型（简化为对角尺度 + 零偏）：

```
a_cal_i = (a_raw_i - bias_i) * scale_i      // i ∈ {x,y,z}
```

**六面法**：把模块六个朝向（±X / ±Y / ±Z 各朝上）分别静置数秒，记录每面读数。理想下朝上轴读 +1g、朝下读 −1g、其余两轴≈0。逐轴解：

```
bias_i  = (a_i⁺ + a_i⁻) / 2           // 正反两面均值 = 零偏
scale_i = 2g / (a_i⁺ - a_i⁻)          // 正反两面跨度 = 尺度
```

- **验收**：标定后任意静止姿态下合加速度模长 `|a|` 应 ≈ 1g（残差 < 数十 mg）。
- **MCU 落地**：离线算出 6 个常数写进 `.h`，运行期只做 `(raw - bias) * scale`，禁止把算好的小数硬写（理由同检查项 6 角速度换算）。

## 检查项 8：磁力计椭球标定（硬铁 / 软铁）

未标定时把 `(mx,my,mz)` 三维点画出来是一个**偏移 + 拉伸的椭球**，而不是以原点为心的球——直接拿去算航向必偏。

- **硬铁误差**：椭球**中心偏移**（车上永磁体/电流产生的常值磁偏置）。
- **软铁误差**：椭球**轴拉伸 / 倾斜**（铁磁材料对外磁场的线性畸变）。

校正模型：

```
m_cal = A · (m_raw - c)        // c=椭球中心(硬铁)，A=变换矩阵(软铁)，校正后落到单位球
```

**流程**：模块在空间各方向缓慢转动画"8 字"采几百~上千点 → 椭球拟合得 `c` 与 `A`（离线用 MATLAB / Python / `magneto` 工具）→ 运行期只做一次减偏 + 3×3 矩阵乘。

> ⚠ 必须**远离电机 / 电池 / 铁磁结构**采点，且**装机后整车重标**——车体磁环境一变，旧参数即失效。无良好磁环境时，宁可不用磁力计 yaw（回到 `mahony-ahrs-reference.md` 的"无磁 yaw 必漂"）。

## 检查项 9：陀螺 Allan 方差（噪声参数标定，给 Kalman 的 Q 提供依据）

EKF/ESKF 的过程噪声 `Q` 不该拍脑袋，其物理来源就是 Allan 方差分解出的噪声系数。

- **采集**：上电预热后**整机静止** ≥ 1–2 小时，高采样率记录陀螺原始角速度（加计同理可得 VRW）。
- **计算**：算 Allan 偏差 `σ(τ)` 对平均时间 `τ` 作 log-log 曲线，按斜率读参数：

| 曲线段 | 斜率 | 物理量 | 折算到 |
|---|---|---|---|
| 短 τ 下降段 | **−1/2** | 角度随机游走 ARW（白噪声） | `Q` 的角速度过程噪声 |
| 谷底 | **0** | 零偏不稳定度（Flicker / bias instability） | bias 建模下限 |
| 长 τ 上升段 | **+1/2** | 速率随机游走 RRW | `bg` 零偏随机游走过程噪声 |

> 把 ARW / RRW 折算到离散 `Q`，直接接 `.auto-embedded/refs/imu-wheel-ekf-fusion.md` 的"Q/R 怎么定"。这一步是 `coding-standards.md` §4.1"参数带来源"在惯导噪声上的物理兑现。

## 快速排查流程

当 IMU 姿态解算出现问题时，按以下顺序排查：

```
1. 轴匹配  → atan2 的轴和 gyro 的轴是否对应？（最常见错误）
2. 量程    → 是否饱和？串口打印原始值，快速转动时是否卡在 ±32768？
3. 灵敏度  → 除数是否与量程匹配？改了量程是否全局更新了灵敏度？
4. DLPF    → 延迟是否超过控制周期的 1/5？
5. 滤波系数 → Alpha 和 dt 是否匹配？时间常数是否合理？
6. I2C     → 读取是否成功？是否有丢帧？
7. 符号    → 陀螺仪积分方向是否与加速度计角度方向一致？
8. 零偏    → 静止时 yaw 是否缓慢漂？换算用公式还是硬编码常数？死区是否吞了低速旋转？
```
