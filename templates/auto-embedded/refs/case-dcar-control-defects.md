# 案例复盘（反例）：四轮全向小车惯导/控制方案缺陷

> 本文件在以下情形按需加载：
> - 你打算照搬某个**电赛小车**的"惯导 / 里程计 / 锁头 / 电机控制"代码作为模板
> - 评审一份四轮底盘固件，想知道"哪些是工程妥协、不能当标准"
>
> 这是一篇**反例**：记录一个能在赛场跑通、但**不应被当作通用模板**的方案（代号 Dcar，STM32F407 四轮全向底盘）。
> 正面做法见 `.auto-embedded/refs/omni-wheel-odometry.md`。与 `control-loop-sign-debug.md` 的"真实案例归档"互补：那篇是符号 bug 复盘，本篇是**架构/标定妥协**复盘。

---

## 为什么单独存一篇反例

赛场代码的最大风险不是"跑不通"，而是**它跑通了**——于是被后人当成"参考实现"复制，把一堆只在那块板子/那场比赛成立的妥协，扩散成"标准做法"。本篇把这些妥协逐条拆开，标注**为什么是妥协**和**正确做法链接**，作为复制前的体检表。

---

## 它做对的（少数值得借鉴的）

先肯定：这套代码工程纪律不差，下面这些可以学：

- **严格分层 + 级联控制**：位置环 → 航向环 → 运动解耦 → 速度环 → 编码器反馈，每层职责单一、单位明确。
- **软硬分工**：对时序敏感的速度环放硬件定时器中断（严格周期），对时序不敏感的决策层放主循环软件分频。
- **非阻塞初始化状态机**：IMU 上电初始化用状态机推进，不 `HAL_Delay` 卡主循环，带错误态自动重试。
- **anti-windup 习惯**：PID 带积分限幅；停车/模式切换/目标突变处复位积分。
- **编码器回绕处理**：`(int16_t)(now-last)` 一行解决 16 位跨零。

> 但"做对的"集中在**架构骨架**；下面的缺陷集中在**惯导算法与标定**——而后者恰恰是最容易被照抄的部分。

---

## 缺陷清单（核心：复制前逐条对照）

### 1. yaw = 纯 Z 轴陀螺积分 + 角速度死区抑漂

**现象**：无磁力计，yaw 靠陀螺 Z 轴积分；用"角速度小于某阈值就不积分"的死区抑制零漂；代码注释自认"yaw 长期必漂"。

**为什么是妥协**：
- 角速度死区会**吞掉真实的低速旋转**——车真的在慢慢转时，转速落在死区内被当成 0，制造**系统性航向误差**（不是随机噪声，是单向偏）。
- 死区掩盖的是**陀螺零偏**，而零偏是可以标定掉的，不该用"装看不见"来处理。

**正确做法**：用上电零偏标定**替代**死区抑漂，死区仅作标定后的兜底噪声门限并注释其"吞低速旋转"副作用——完整流程（静止检测 / 求 bias / 重标周期）见 `imu-gyroscope-checklist.md` 检查项 6；要绝对 yaw 上 9 轴磁力计/视觉，见 `mahony-ahrs-reference.md`。

### 2. 里程计 `dt = 0.01f` 硬编码

**现象**：积分周期直接写死 0.01s。

**为什么是妥协**：调度频率一改、或定时器有 jitter，积分就系统性偏，而且**看不见**（没有任何运行期校验）。

**正确做法**：`dt` 由定时器 reload 计算的命名常量；高要求场景运行期监测实际调用间隔，或对周期 `static_assert`。见 `omni-wheel-odometry.md`、`coding-standards.md` §4.1。

### 3. 定点用"匀速指向目标"，不是位置闭环调速

**现象**：相对位移任务里，朝目标点方向给一个**恒定速度**，到达死区内才停。

**为什么是妥协**：末端不减速 → 易冲过头/震荡；对距离误差无自适应；到达判定靠死区半径，死区大了精度差、小了停不下来。

**正确做法**：位置环输出**限幅/梯形规划**的速度（远快近慢），或标准位置 PID；末端减速段独立处理。

### 4. 打滑不可观测 → 编码器高估位移

**现象**：位置完全由编码器反解的轮速积分得到，无任何打滑检测。

**为什么是妥协**：高速转向/低摩擦地面打滑时，轮子空转，编码器照常计数，里程计**高估**位移且无从发现。

**正确做法**：多源校验（IMU 加速度 vs 轮速一致性）、打滑检测、外部参考校正。本质见 `omni-wheel-odometry.md` 铁律。

### 5. 魔法数字遍布（最该警惕）

**现象**：`Kp=60 / Ki=0.7 / 旋转限幅 ±800`、`PWM<60 直接空载`、`yaw 误差死区 0.3°`、`角速度死区 ±3°/s`、`dt=0.01` 等阈值散落、无溯源。

**为什么是妥协**：这些数字和**这块板子的** PWM 分辨率、电池电压、电机/减速比、轮距、控制周期、地面摩擦强耦合。换任何一项就失效；写进"参考代码"会被当成普适常数照抄。

**正确做法**：每个阈值/死区/限幅/增益都是命名常量，注释带**单位 + 来源（实测/派生/手册）+ 测量方法 + 适用硬件**。规范见 `coding-standards.md` §4.1。

### 6. 手写浮点 PID/Mahony + 硬编码增益，与"MATLAB 出系数"路线冲突

**现象**：控制/估计算法在 C 里手写，增益硬编码在源码里。

**为什么是妥协**：与本知识库主线（`embedded-matlab` 算系数 → 导出 `.h` → C 只消费）相悖；增益不可复现、不可在不改源码下整定。注意：简单 PID **服务模块**本身是允许的（见 `embedded-alg` 的 `svc_pid.c`），**红线是硬编码魔法增益 + 重写复杂估计器**，不是"存在 PID 环"。

**正确做法**：增益来自配置/`.h`；复杂估计器（Kalman/LQR/FFT）用 `embedded-matlab` 产物。见 `lqr-example-segway.md`。

## 一句话结论

Dcar 的**架构骨架**值得学（分层、级联、软硬分工、非阻塞初始化）；它的**惯导与标定**是短赛程妥协——纯陀螺 yaw、硬编码 dt、死区抑漂、匀速定点、魔法数字。**可作演示，不可作鲁棒定位/通用控制模板照抄。** 复制任意一段前，回到本表对应条目确认你没把妥协一起搬走。
