## 铁律

**轮式里程计是航位推算（dead reckoning），不是融合定位。**

它把"轮子转了多少 + 车头朝哪"积分出位置。三个误差源**不可观测、随时间/路程单调累积、且无法自我纠正**：

1. **yaw 漂移**：无磁力计/视觉/外部参考时，yaw 来自陀螺积分，长期必漂（见 `mahony-ahrs-reference.md`）。yaw 一旦偏，之后所有位移都投影到错误方向。
2. **打滑 / 空转**：轮子打滑时编码器照常计数，里程计**高估**位移；轮子被卡住时**低估**。编码器测的是"轮子转角"，不是"车体位移"。
3. **标定误差**：轮径、轴距、`COUNTS_PER_MM` 的标定误差会按比例放大到每一段位移。

**结论**：它适合**短程、低打滑、有闭环校正机会**的场景（电赛定点、相对位移）。把它当鲁棒绝对定位用，或给它"通用定位方案"地位，是错的。需要长期绝对位姿 → 上 EKF 融合（轮速+IMU+磁/视觉/UWB），见文末"何时升级"。

## 第 0 步：先把坐标系和单位钉死（最高优先级）

运动学公式的符号**完全取决于约定**。动手前必须在一处注释里写死下面 5 项，否则正解、反解、里程计三处迟早打架：

| 约定项 | 必须明确 | 示例 |
|---|---|---|
| 车体轴 | +X 指哪、+Y 指哪 | +Y 车头前方、+X 车体右侧 |
| yaw 正方向 | CW 为正 or CCW 为正 | yaw 增大 = 车头顺时针（与 IMU 一致） |
| 轮编号与物理位置 | 每个轮 ID ↔ 安装位置 | s1=左前 s2=右前 s3=右后 s4=左后 |
| 轮速正方向 | 轮速 > 0 对应车"前进"还是"后退" | s_i > 0 → 该轮驱动车体 +Y |
| 单位 | 车体速度、轮速、角速度各自单位 | vx/vy [mm/s]、ωz [rad/s] 或 w [mm/s 等效轮缘] |

> ⚠ 单位陷阱：很多电赛代码把旋转量 `w` 写成"等效轮缘线速度 [mm/s]"而不是真正的 `ωz [rad/s]`，从而把轴距 `(lx+ly)` 和轮径折叠进去。这能少写常数，但**该 `w` 不能直接当角速度用**（不能拿去积分 yaw、不能和 IMU 角速度比较）。要省就省得明白，并在注释里标注。

## 运动学：正解（解耦）与反解（里程计）

### 完整形式（mecanum，含几何量，推荐）

设车体速度 `vx, vy [m/s]`、`ωz [rad/s]`，轮径 `r [m]`，半轴距 `lx`、半轮距 `ly [m]`。各轮**角速度** `ωi [rad/s]`：

```
正解（车体 → 四轮，inverse kinematics）：
  ω_LF = (1/r) ( vy + vx + (lx+ly)·ωz )
  ω_RF = (1/r) ( vy − vx − (lx+ly)·ωz )
  ω_RR = (1/r) ( vy + vx − (lx+ly)·ωz )
  ω_LR = (1/r) ( vy − vx + (lx+ly)·ωz )
```

> 上式的 ± 号取决于第 0 步的轴/yaw 约定与滚子朝向（X 型 or O 型布局），**不要照抄符号**，按 `control-loop-sign-debug.md` 逐轮验证。

反解（里程计，forward kinematics）是正解矩阵的**最小二乘伪逆**（4 轮 3 自由度，超定）：

```
反解（四轮 → 车体）：
  vy = (r/4)( ω_LF + ω_RF + ω_RR + ω_LR )
  vx = (r/4)( ω_LF − ω_RF + ω_RR − ω_LR )
  ωz = (r / 4(lx+ly))( ω_LF − ω_RF − ω_RR + ω_LR )
```

### 简化形式（"等效轮缘速度"，电赛常见）

若直接用**轮缘线速度** `s_i [mm/s] = r·ω_i` 且把 `w` 当等效轮缘量（折叠掉 `r` 和 `(lx+ly)`）：

```
正解：  s1 = vy + vx + w     反解：  vy = (s1+s2+s3+s4)/4
        s2 = vy − vx − w             vx = (s1−s2+s3−s4)/4
        s3 = vy + vx − w             w  = (s1−s2−s3+s4)/4
        s4 = vy − vx + w
```

**这是上面完整式的特例**，代价是：`w` 不是 `ωz`、丢了轮径/轴距的物理意义、所有量必须同单位。**能用，但要清楚它折叠了什么**——否则换底盘几何时无从校正。

### 正反解必须互为逆运算（最易踩的坑）

解耦正解（控制下发）和里程计反解（位置估计）**必须用同一套符号约定，互为逆运算**。任何一处符号、轮编号、安装方向变了，**两处都要同步改**。它们不一致时的典型症状：车能正常走，但里程计反向漂、转向相反。验证法见
`control-loop-sign-debug.md` 的"多轮底盘电机/编码器极性矩阵验证"小节。

## 里程计积分：车体系 → 世界系

```
每个控制周期（dt 来自定时器配置，禁止硬编码——见下）：
  1) 读四轮编码器增量 → 轮速 s_i → 反解 vx, vy（车体系）
  2) θ = yaw（来自 IMU，单位 rad）
  3) 旋转到世界系并积分：
       dx = ( vx·cosθ + vy·sinθ )·dt
       dy = (−vx·sinθ + vy·cosθ )·dt
       x += dx;  y += dy
```

> **里程计旋转式 = R(θ)ᵀ（CW 约定）**：上式 `dx=(vx·cosθ+vy·sinθ)dt、dy=(−vx·sinθ+vy·cosθ)dt` 对应 `R=[[cosθ,sinθ],[−sinθ,cosθ]]=R(θ)ᵀ`，是 **CW 旋转**，仅当 yaw 取第 0 步示例的 **CW 为正** 时自洽。若你的 yaw 是 **CCW 为正**（姿态侧 Mahony / `attitude-init-single-frame.md` 用 ENU/标准 ZYX 即 CCW），改用对照式 `dx=vx·cosθ−vy·sinθ、dy=vx·sinθ+vy·cosθ`。按自身 yaw 符号二选一，并用"直线沿 θ=90° 走一段看 x/y 落点"做一次符号自检（闭合测试也会抓出镜像反向轨迹）。

> **dt 必须来自定时器周期配置，不要写 `dt = 0.01f` 这种裸字面量。** 调度频率一改、或定时器有 jitter，硬编码 dt 会让积分系统性偏。正确：`dt` 由 timer reload 计算得到的命名常量；高要求场景在运行期监测两次调用的实际间隔（jitter），或至少对调度周期加 `static_assert`。阈值/常数的标定溯源规范见 `coding-standards.md` §4.1。

## 航向保持（"锁头"）行为状态机（去参数化模板）

全向底盘平移时，因四轮速度不匹配/打滑，车头会慢慢歪。常用做法：**无主动旋转指令时，自动用 yaw 闭环保持"进入瞬间的航向"**。这是一个可复用的**行为状态机**（与具体 PID 参数无关）：

| 状态 | 进入条件 | 动作 |
|---|---|---|
| 退出锁头 | 无平移且无旋转指令 | 停车，`target_yaw ← 当前 yaw`，复位 yaw PID |
| 用户接管 | 有主动旋转指令 | 不保持，`target_yaw ← 当前 yaw`，复位 PID（防积分残留） |
| 保持航向 | 有平移、无旋转指令 | 首次进入锁存 `target_yaw`；之后 `w ← PID(target_yaw − yaw)` |

> ⚠ 本状态机里的 PID 增益、误差死区、旋转限幅**都必须按你的底盘标定**，禁止照抄任何工程的具体数字（那是别人某次调参的结果，和惯量/摩擦/电池电压/控制周期强耦合）。标定溯源见 `coding-standards.md` §4.1。yaw 与底盘旋转方向相反时，翻 PID 输出符号或 yaw 符号，按 `control-loop-sign-debug.md` 验证。

## 编码器测速：16 位计数器跨零回绕（可直接抄，附前提）

正交编码器接定时器编码器模式时，`CNT` 是 16 位会回绕。求每周期增量：

```c
/* 前提条件（全部满足才成立）：
 *   1) 单个采样周期内 |增量| < 32768（否则无法区分高速与反向回绕）→ 采样要够快
 *   2) CNT 读取是单次原子读（16/32 位对齐读）
 *   3) 单位换算 COUNTS_PER_MM 由 编码器线数×4倍频×减速比 / (π×轮径) 推导，不写裸常数 */
static inline int16_t enc_delta(uint16_t now, uint16_t last) {
    return (int16_t)(now - last);   /* 无符号减法自然回绕 + 强转有符号，跨零自动正确 */
}
/* 轮速 = enc_delta × 1000 / sample_ms [count/s]；线速度 [mm/s] = (count/s) / COUNTS_PER_MM */
```

## 验证方法（落地前必做）

1. **逐轮极性**：单独给某轮 +指令，确认该轮按"约定正方向"转、且其编码器读数**增大**（见 `control-loop-sign-debug.md`）。
2. **正反解往返**：给定 `(vx,vy,w)` → 正解 → 反解，应还原回 `(vx,vy,w)`（数值往返测试）。
3. **直线测距**：让车走已知直线距离，比对里程计 `Δ` 与卷尺，标定 `COUNTS_PER_MM`。
4. **闭合测试**：走一圈回到起点，看里程计 `(x,y)` 闭合误差和 yaw 净变化（应 ≈0）；闭合误差大 → 查 yaw 漂移 / 打滑 / 标定。
5. **绝对定位误差**：走已知折线/方阵、经过若干已知锚点，逐点比对里程计 `(x,y)` 与卷尺/激光实测（不是只看终点闭合——闭合会**掩盖"沿途漂出去又漂回来"的中途偏差**）。
6. **漂移率**：固定速度直线/方阵跑 `T` 秒或 `D` 米，记 `位置误差/路程 [%]` 与 `yaw 漂移率 [°/min]`，作为"是否需升 ESKF/加绝对观测"的跨方案判据。

## 何时升级（超出本方案能力）

出现以下需求，就别再加补丁，直接上**多源融合**（EKF/UKF：轮速 + IMU + 磁力计/视觉/UWB/激光）：

- 需要**长期绝对位姿**（分钟级以上不重置）
- 工况**打滑严重**（高速转向、地面摩擦低）
- 需要**绝对 yaw**（仅靠陀螺积分不够）

> 升 ESKF 后，**差速/阿克曼**车可几乎免费加 **NHC 非完整性约束**（车体侧向速度≈0）持续压横向漂；**全向/麦轮**可侧移、NHC 不适用（详见 `imu-wheel-ekf-fusion.md`）。纯航位推算阶段也可借 NHC 思想做合理性校验：侧向速度异常 → 疑似打滑。

融合后轮速里程计退化为"高频预测源"，由外部观测做校正——这才是鲁棒定位的正确形态。
