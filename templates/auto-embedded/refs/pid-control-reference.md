# PID 控制参考（位置式/增量式 · 抗饱和 · 微分处理 · 离散化时序 · 整定 · 串级）

> 本文件在以下情形按需加载：
> - 要写/调一个 PID（速度环、位置环、温度、平衡、任何闭环）
> - 纠结位置式 vs 增量式、积分饱和、微分噪声、采样周期、怎么整定
>
> 这是控制簇的**基础地基**：FOC 电流/速度/位置环、平衡车、路径跟踪都建在 PID 上。

## 铁律

**PID 是放大器，不是方向修正器：符号错了，调参全在给错误方向加油（见 `control-loop-sign-debug.md`）。先验证方向，再调参；先内环后外环、先 P 再 I 最后 D。**

- 增益 `Kp/Ki/Kd`、限幅、死区、采样周期 `Ts` **都是标定参数**，带单位+来源+测量方法+适用硬件，禁裸魔法数字。
- **`dt` 必须来自定时器周期，不要写 `dt=0.01f` 裸字面量**（调度一改就系统性偏，呼应 `omni-wheel-odometry.md` 反例）。

## 连续 → 离散：位置式 vs 增量式

连续：`u(t) = Kp·e + Ki·∫e dt + Kd·de/dt`，`e = 目标 − 反馈`。

**位置式**（每拍算输出绝对值）：
```
u[k] = Kp·e[k] + Ki·Ts·Σe + Kd·(e[k]−e[k−1])/Ts
```
**增量式**（每拍算输出增量，对累加器/PWM 寄存器做 `+=`）：
```
Δu[k] = Kp·(e[k]−e[k−1]) + Ki·Ts·e[k] + (Kd/Ts)·(e[k]−2e[k−1]+e[k−2])
u[k]  = u[k−1] + Δu[k]
```

| | 位置式 | 增量式 |
|---|---|---|
| 积分 | 显式累加器（要主动抗饱和） | 隐含在 `u[k−1]`，**天然限幅**（对 u 限幅即限了积分） |
| 切换/复位 | 需手动清积分防跳变 | 无显式积分，模式切换更平滑 |
| 故障/丢拍 | 单拍错只影响该拍 | 单拍错会累进 `u`（要防护） |
| 何时用 | 通用、要直接限输出、要积分分离 | 输出是"增量"语义（步进/阀门）、要天然抗饱和 |

> 工程里**位置式总体更常用**（要绝对输出、好做积分分离/前馈）；增量式在"天然抗饱和/平滑切换"场景有优势。

---

## 抗积分饱和（必做）

输出顶到限幅后误差仍让积分继续"蓄力" → 退饱和迟滞、超调。三招（位置式）：

- **钳位 Clamping**：`u` 饱和且积分会让其更饱和时，**冻结积分**（不再累加该方向）。最简，够用。
- **反算 Back-calculation**：`I += Ki·Ts·e + Kb·(u_sat − u_unsat)`，把饱和差按增益 `Kb` 退回积分，退饱和更平滑。
- **积分分离**：`|e|` 大时关 I（只 PD 快速逼近）、`|e|` 小时再开 I 消静差，避免大误差段积分爆掉。
- 上电/模式切换/目标突变时**复位积分**。

---

## 微分项处理（别让 D 放大噪声）

- **微分先行 / 作用于测量**：`D = −Kd·(y[k]−y[k−1])/Ts`（对反馈微分、取负），避免目标阶跃时微分尖峰（"微分冲击"）。
- **不完全微分（低通）**：`D[k] = α·D[k−1] + (1−α)·Kd·(...)`，`α∈[0,1]` 由截止频率定。裸微分会把采样噪声放大成抖动。
- 一阶对象（如 FOC 电流环）**用 PI 不用 D**（D 无收益、徒增噪声，见 `foc-current-loop.md`）。

---

## 前馈（FF）

`u = PID(e) + u_ff`。把已知部分直接补上、让 PID 只补残差：速度前馈 `Kv·v*`、加速度前馈 `Ka·a*`（由运动规划给，见 `motion-profiling-trajectory.md`）、重力/摩擦补偿。**前馈降滞后比硬拉 Kp 更稳**。

---

## 离散化与时序

- **采样周期 `Ts`**：采样率 `fs ≥ 4–10×` 闭环带宽（即 `Ts ≤ 1/(4~10·带宽)`；4× 为经验下限、相位裕度偏紧，常取 ≥10×）。太慢→相位滞后失稳，太快→噪声/算力。
- **离散化**：积分用后向欧拉（稳、有半拍滞后）或 Tustin（相位更好）；`I[k]=I[k−1]+Ki·Ts·e[k]`（后向欧拉）。
- **计算延迟**：ISR 里"采样→算→输出"有一拍延迟，高带宽环要建模/补偿。
- `dt` 由 timer reload 得到的命名常量；高要求场景运行期监测实际间隔（jitter）。
- ADC 抗混叠：采样前模拟/数字低通，避免高频折叠进控制带宽。

---

## 整定方法

- **手调（最常用）**：`I=D=0`，加大 `Kp` 到轻微振荡退半；加 `Ki` 消静差（配抗饱和）；必要时加 `Kd`（配低通）抑超调。**先内环后外环**（电机：先电流/直立，再速度，最后位置）。
- **Ziegler-Nichols（临界比例度）**：只留 P，加大到等幅振荡，记临界增益 `Ku`、周期 `Tu`：经典 PID → `Kp=0.6Ku, Ti=0.5Tu, Td=0.125Tu`（即 `Ki=Kp/Ti, Kd=Kp·Td`）。给个起点，仍需微调。
- **继电(relay)自整定**：用继电环节激起极限环自动测 `Ku/Tu`，适合在线自整定。
- 模型已知（如 FOC 电流环）→ **解析整定**（零极点对消，见 `foc-current-loop.md`），优于试凑。

## 串级（cascade）要点

- **内环带宽 ≥ 外环 5–10 倍**，外环才能把内环当"理想跟随"（详见 `foc-speed-position-loop.md`、`balance-car-cascade-control.md`）。
- 内环输出限幅 = 外环的**天然限速/限矩点**（如速度环限幅 = 转矩上限）。
- 串级可把**微分放在前馈路径**（作用于无噪声的参考），反馈路径保 PI——比单级 PID 把 D 加在带噪误差上更干净。

## 代码骨架（位置式 + 抗饱和 + 微分低通）

```c
typedef struct { float Kp,Ki,Kd, Ts, i, d, y_prev, out_min,out_max, d_alpha; } PID;

float pid_step(PID* p, float target, float meas) {
    float e = target - meas;
    // 积分（后向欧拉）+ 钳位抗饱和
    float i_new = p->i + p->Ki * p->Ts * e;
    // 微分作用于测量 + 一阶低通（不完全微分）
    float d_raw = -p->Kd * (meas - p->y_prev) / p->Ts;
    p->d = p->d_alpha * p->d + (1.0f - p->d_alpha) * d_raw;
    float u = p->Kp * e + i_new + p->d;
    // 限幅 + 仅在不进一步饱和时才接受新积分（clamping）
    if (u > p->out_max)      { u = p->out_max; if (e < 0) p->i = i_new; }
    else if (u < p->out_min) { u = p->out_min; if (e > 0) p->i = i_new; }
    else                     { p->i = i_new; }
    p->y_prev = meas;
    return u;
}
```

## 常见陷阱

| 陷阱 | 说明 |
|---|---|
| 先调参不验方向 | 符号错时调参全错（`control-loop-sign-debug.md`）|
| 无抗饱和 | 积分爆掉、退饱和迟滞、超调 |
| 裸微分 | 放大采样噪声→抖动；用微分先行+低通 |
| `dt` 硬编码 | 调度变就系统性偏 |
| 采样太慢 | 相位滞后失稳；`Ts ≤ 1/(4~10·带宽)` |
| 目标阶跃微分冲击 | 微分作用于测量而非误差 |
| 串级带宽不分离 | 内外环打架振荡 |
