# 简单执行器控制（有刷 DC · 方波六步 BLDC · 舵机 · 步进）—— FOC 之外的常用选择

> 本文件在以下情形按需加载：
> - 不需要 FOC 那么重，要驱动有刷直流电机 / 方波 BLDC / 舵机 / 步进电机
> - 纠结"上不上 FOC"，或要给这些执行器做闭环/定位
>
> FOC（PMSM 高性能）见 `.auto-embedded/refs/foc-control-overview.md`；闭环用的 PID 见 `pid-control-reference.md`；符号见 `control-loop-sign-debug.md`。

---

## 铁律

**先按需求选最简执行器：能用有刷 PID / 舵机 / 步进解决的，别硬上 FOC。简单不等于差——选错复杂度才是反模式。**

| 执行器 | 控制核心 | 闭环 | 何时选 |
|---|---|---|---|
| **有刷直流** | H 桥 PWM 调压 + 方向 | 编码器 → 速度/位置 PID | 最通用、低成本、电赛主力 |
| **方波六步 BLDC** | 霍尔扇区→两相通电（六拍） | 霍尔 + 速度 PID | BLDC 但不要 FOC 的平滑，省算力 |
| **舵机（RC servo）** | 50Hz PWM 脉宽↔角度 | 内部自带（开环给角度） | 有限角度定位（转向/云台关节） |
| **步进** | 按相序脉冲走步 | 开环计数 / 闭环加编码器 | 开环精确定位、低速大扭矩 |
| FOC（PMSM） | dq + SVPWM | 电流/速度/位置 | 要平滑转矩、宽调速、精确转矩（见 foc 簇） |

## 有刷直流电机（最常用）

- **驱动**：全 H 桥 IC（如 TB6612/DRV8833），或两片半桥拼 H 桥（如 BTN8982 → IBT-2/BTS7960 模块）；两路 PWM 或 PWM+方向；互补桥配死区。
- **测速**：正交编码器（定时器编码器模式，16 位回绕用 `(int16_t)(now-last)`，见 `omni-wheel-odometry.md`）。
- **速度环**：`PWM = PID_speed(v* − v)`（PI 为主，速度噪声大不加 D）。
- **位置环**：外层 `v* = PID_pos(θ* − θ)`，串级（见 `pid-control-reference.md` 串级 + 带宽分离）；或单环位置 PD + 速度前馈。
- **陷阱**：PWM 最小占空比死区（低占空电机不转，需补偿或前馈）；H 桥死区；方向切换瞬间电流冲击；编码器极性与 PWM 极性要一致（`control-loop-sign-debug.md`）。

## 方波六步换向 BLDC（霍尔）

无需坐标变换：任意时刻**两相通电、一相悬空**，按转子位置切换六个通电状态（六拍）。

```
霍尔(HU,HV,HW) 3 位 → 6 个有效扇区(2 个无效) → 查表得该扇区的上/下桥通断组合
例：扇区1 → A+ B−（C 悬空）；转过 60° 电角度切到下一拍
PWM 调上桥占空比 = 调电压 = 调转速；外层 PID 速度环
```

- **建换向表**：慢转一圈记录每个霍尔组合对应的通电状态（注意厂商相位/接线）。错位 → 倒转/抖动/堵转。
- **无感方波**：用悬空相**反电动势过零检测**（ZCD）替代霍尔（零速无法启动，需开环强拖起步）。
- **vs FOC**：实现简单、算力低；代价是换相处**转矩脉动**、噪声大。要平滑/精确转矩 → 上 FOC（`foc-control-overview.md`）。

## 舵机（RC servo / 数字舵机 / 总线舵机）

- **标准 PWM**：周期 20ms（50Hz），脉宽 **500–2500µs ↔ 角度**（常见 1000–2000µs↔0–180°，厂商不一）。占空比定时器直接生成，**无需自建 PID**（舵机内部已闭环）。
- **标定**：实测每只舵机的**行程端点 + 中位 + 死区**（机械限位别打死，留余量防堵转烧机）。
- **数字舵机**：更高刷新率/分辨率、响应快；**总线舵机**（如串行总线）可读回位置/温度、菊花链，适合多关节。
- **陷阱**：脉宽超范围打死堵转→烧；多舵机共电源要够流（峰值大）；模拟舵机抖动用数字舵机或滤波目标。

## 步进电机

- **走步**：两相（或三相）按相序脉冲走步。**整步 / 半步 / 细分（微步）**：细分用正余弦电流插值出中间位置，提升分辨率与平滑度、减振。
- **开环定位**：脉冲数 = 位置（不丢步前提下），最廉价的"精确定位"——但**无反馈、丢步不可知**。
- **丢步**：负载超过力矩、加速过猛、共振时会丢步且静默。对策：留力矩余量、用运动规划（梯形/S曲线，见 `motion-profiling-trajectory.md`）限加速、避开共振区。
- **闭环步进**：加编码器，丢步时补偿/告警；高级做法是"位置环 + 开环 FOC"（两相 SVPWM，省电流采样）——介于步进与伺服之间。
- **细分档位经验**：低速高精开环可高细分（如 32–64）；高速细分对平滑收益小且**高速失矩**；闭环则整步/低细分即可（编码器保精度，细分主要减振）。
- **陷阱**：高速力矩跌落、中频共振、细分越高单步扭矩越小、发热（恒流驱动）。

## 通用陷阱

| 陷阱 | 说明 |
|---|---|
| 硬上 FOC | 简单定位/调速用有刷/步进/舵机即可，别过度设计 |
| 极性不一致 | PWM/方向/编码器/霍尔极性要单点归一（`control-loop-sign-debug.md`）|
| 无运动规划 | 步进/有刷阶跃指令→丢步/冲过头（`motion-profiling-trajectory.md`）|
| 舵机打死限位 | 堵转烧机；标定留余量 |
| 忽略最小占空死区 | 有刷低速不转、控制非线性 |
